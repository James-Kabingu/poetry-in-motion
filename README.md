## Rules of Use

1. Do not treat mock APIs as production claims until tags move from [IMPLEMENTED] to [VALIDATED] against real data.
2. Add new research under the numbered folders inside Research/, not the repository root.
3. Align code changes with Research/01-global/LANDSCAPE.md Part 3.2 (inherit and reject table) before expanding scope.
4. Any strategy change must be reflected in Research/06-mvp/MVP.md before the corresponding code is written.
5. Research/05-critical-review/CRITICAL-REVIEW.md is reviewed quarterly. It takes precedence over foundation documents where they conflict.
